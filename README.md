# Cybersecurity Governance and Security Policy Framework for CareFirst Medical Group

## Master’s Cybersecurity Capstone Project

This repository presents my Master’s-level cybersecurity capstone project, developed as part of my graduate studies at Strayer University.

The project focuses on designing a comprehensive cybersecurity governance, risk management, and security policy framework for **CareFirst Medical Group**, a fictional nonprofit healthcare organization operating in a hybrid technology environment.

> **Academic Disclaimer:** CareFirst Medical Group is a fictional organization developed exclusively for academic purposes. This repository does not contain real patient information, protected health information, confidential business data, or production credentials.

## Project Overview

Healthcare organizations manage highly sensitive data while relying on interconnected clinical systems, cloud services, telemedicine platforms, remote access technologies, and third-party vendors.

This project evaluates the cybersecurity risks affecting a fictional healthcare organization and proposes a structured security governance and policy framework aligned with recognized industry standards.

## Project Objectives

- Establish cybersecurity governance and accountability.
- Identify organizational risks and security control gaps.
- Strengthen identity and access management.
- Improve network and endpoint security.
- Protect sensitive healthcare information.
- Develop incident response and ransomware preparedness procedures.
- Address third-party and supply-chain risk.
- Establish artificial intelligence governance policies.
- Support compliance, auditing, and continuous improvement.

## Frameworks and Standards

The project incorporates guidance from:

- NIST Cybersecurity Framework 2.0
- NIST SP 800-53
- NIST SP 800-63B
- NIST SP 800-207
- NIST AI Risk Management Framework
- HIPAA Security Rule
- ISO/IEC 27001
- GDPR
- Zero Trust Architecture principles
- CISA healthcare cybersecurity guidance

## Artificial Intelligence Governance

The project includes three primary AI governance areas:

1. Clinical Oversight and Accountability
2. Data Governance and Privacy
3. AI Model Lifecycle Management

These policies address human oversight, privacy, model monitoring, security testing, bias risk, data authorization, and responsible AI use.
